## 🔭 本周一个大问题

### OpenAI 和 Anthropic 同一天互砸补贴——AI 编程工具真的会像外卖大战一样「烧出」赢家吗？

5 月 14 日这天很有意思。

OpenAI 宣布：从 Claude Code 迁移到 Codex 的企业/团队，免费用两个月（价值 $400）。几个小时后，Anthropic 反手宣布：Claude Code 付费用户月度额度直接加量 50%。

3 小时内，2000 名开发者联系 OpenAI 咨询迁移。Sam Altman 亲自站台。Anthropic 方面也毫不示弱，同时推出 Opus 4.7 Fast 模式。

这场面，太像 2015 年的外卖大战了——滴滴和快的互砸补贴，最后合并了。

**但 AI 编程工具的竞争逻辑和外卖不同。**

外卖平台的核心壁垒是网络效应——骑手越多、商家越多、用户越多，飞轮转起来。但编程工具的壁垒是**工作流锁定**：一旦开发者把日常流程搭建在某个工具上（配置 prompt、建立 memory、训练 skill），迁移成本就指数上升。

这就是为什么 OpenAI 挖的不是「用户」而是「迁移窗口期」——他们赌的是，两个月足够让开发者把工作流搬完，之后就走不了了。Anthropic 加量 50% 的防守逻辑也很清楚：提高在位优势，让你没有动力去折腾迁移。

从万维钢式科学思维看这个现象：**这是典型的「转换成本博弈」。** 经济学家 Paul Klempner 早就证明过——在高转换成本市场中，先行者对存量用户的定价会逐步提高，而挑战者对新用户的补贴力度也会逐步加码。最终的均衡不是一家独大，而是市场分裂成两个高度粘性的用户群。

对我们这种创意/设计团队来说，这意味着什么？

**现在是薅羊毛的黄金窗口。** 两个月后这些补贴大概率消失，但两边给出的永久额度提升会留下来。建议做法：同时申请 Codex 免费试用和 Claude Code 加量，用两个月时间 A/B 测试到底谁更适合你的工作流——然后在补贴结束前做出选择。

而更深层的思考是：当 AI 编程工具变成像 IDE 一样的日常基础设施时，**你选择的不只是一个工具，而是一个 AI 思维方式。** Codex 更快更激进，Claude Code 更深更稳。这个选择会反向塑造你的编程习惯和项目架构。

就像选 Vim 还是 Emacs——一旦肌肉记忆形成，你就再也不想换了。
