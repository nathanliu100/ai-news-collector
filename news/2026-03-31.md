## 💡 AI 效率技巧

### 🟢 [Context Engineering 正式成为 AI 应用的核心方法论](https://www.anthropic.com/engineering/effective-context-engineering-for-ai-agents)
**来源**：Anthropic 官方 / GitHub Blog [🏢 官方] | **时间**：2026-03 持续更新
Anthropic 和 GitHub 都发布了关于 Context Engineering 的深度指南。核心思路：**从"写好一个 prompt"转向"设计整个上下文系统"**，包括记忆管理、工具调用编排、多 Agent 协作架构。GitHub 提出的三层框架：Agentic Primitives（基本能力）→ Context Engineering（上下文设计）→ Reliable Workflows（可靠工作流），正在成为行业标准方法论。

### 🟢 [IBM 发布 2026 Prompt Engineering 终极指南](https://www.ibm.com/think/prompt-engineering)
**来源**：IBM [🏢 官方] | **时间**：2026-03-29
IBM 更新其 Prompt Engineering 指南，覆盖 Granite、GPT、Claude 等主流模型。2026 年的关键变化：prompt 不再是"一句话指令"，而是包含角色设定、示例、约束条件、输出格式的**结构化模板系统**。指南中最实用的建议：保持 400ms 以内的响应延迟可维持用户心流状态。
