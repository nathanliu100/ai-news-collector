## 💡 AI Productivity Tips

### 🟢 [Agent Prompt Engineering Best Practices 2026: Treat Prompts Like Code](https://musketeerstech.com/blogs/prompt-engineering-best-practices)
**Source**: Musketeers Tech [📰 Media] [🟨 T2] | **Time**: July 5
Prompt engineering for AI Agents in 2026 has evolved from "copywriting" to "systems engineering." Core practices: (1) Write system prompts as operating manuals, not descriptions; (2) Build 20-50 case eval sets for regression testing; (3) Write tool descriptions as carefully as prompts; (4) Use rolling summaries instead of full history to save 60% tokens; (5) Version control: Git + logging + eval run on every change. Cross-model adaptation differences are only ~10% — 90% of structure transfers across models.

### 🟢 [Three Tricks to Boost Small Models: 7B Model Accuracy Jumps from 52% → 81% via Prompt Optimization](https://www.toutiao.com/article/7657725235718668815)
**Source**: Toutiao [📰 Media] [🟥 T3] | **Time**: July 2
Inspired by HN trending project Micro-Agent: without changing models or adding parameters, collaborative internal prompting pushes small model performance near frontier models. Three tested techniques: (1) Structured decomposition — force 4-segment output (understanding→decomposition→solving→answer), +30% accuracy; (2) Self-consistency — run same question 5 times, take majority vote; (3) Reflective iteration — let model answer first, then self-check and correct. Combined, Qwen 7B accuracy on GSM8K math problems jumped from 52% to 81%.
