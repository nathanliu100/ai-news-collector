### 🔴 [NVIDIA 发布 Nemotron 3 Nano Omni：视觉+语音+语言统一的开源全模态模型](https://blogs.nvidia.com/blog/nemotron-3-nano-omni-multimodal-ai-agents/)
**来源**：NVIDIA 官方博客 [🏢 官方] [🟩 T1] | **时间**：2026-04-28 | ✅ 已验证：NVIDIA 官方博客 + HuggingFace 同步上线 + AWS SageMaker Day-0 可用
NVIDIA 发布开源全模态推理模型 Nemotron 3 Nano Omni，30B 总参数/3B 激活（MoE 架构），将视觉、音频、语言能力统一到单一模型中。在 6 个排行榜上名列前茅，吞吐量比同类开放全模态模型高出 **9 倍**。支持 256K 上下文、1920×1080 原生分辨率 GUI 理解。已在 HuggingFace、OpenRouter、AWS SageMaker 等 25+ 平台上线。定位为 AI Agent 系统的"眼睛和耳朵"，与 Nemotron 3 Super/Ultra 协作构建多 Agent 系统。[💎 部门相关]
<!-- crew:elon -->**开源全模态是正确方向**——但 30B 参数 3B 激活的 MoE，说白了就是用架构巧妙规避算力门槛，让每个开发者都能跑。NVIDIA 在卖铲子的同时开始送铲子了。<!-- /crew -->
<!-- crew:jobs -->9 倍效率提升值得关注，但关键是**体验是否连贯**。多模态不是把三个模型捏在一起，是让用户感觉只跟一个智能体对话。<!-- /crew -->
<!-- crew:wanweigang -->MoE 架构的核心洞见是：**大多数输入只需要一小部分参数就够了**。30B 参数 3B 激活，意味着 90% 的参数在"休息"。这是工程上的贝叶斯最优——用最少的计算覆盖最多的场景。<!-- /crew -->
<!-- crew:tim -->作为内容创作者我最关心的是**它能不能真正理解视频内容**，不是识别画面里有什么，而是理解镜头语言、剪辑节奏。如果能做到这一步，后期工作流会被彻底改写。<!-- /crew -->
<!-- crew:mrbeast -->一个模型就能看、听、读？**这意味着做视频的人终于不用在五个工具之间切换了**。把它接到剪辑流程里，光是字幕+画面校对就能省掉一个人的工作量。<!-- /crew -->

### 🔴 [OpenAI × Microsoft 重写合作协议：AGI 条款废除、排他权取消](https://blogs.microsoft.com/blog/2026/04/27/the-next-phase-of-the-microsoft-openai-partnership/)
**来源**：Microsoft 官方博客 + The Verge + VentureBeat [🏢 官方] [🟩 T1] | **时间**：2026-04-27 | ✅ 已验证：Microsoft 官方博客 + The Verge + TechCrunch + VentureBeat 四家独立报道
这可能是 2026 年 AI 产业最重要的结构性变化之一。Microsoft 和 OpenAI 4/27 官宣重签合作协议，核心变化：**(1)** 废除 Azure 排他权——OpenAI 可自由在 AWS、Google Cloud、Oracle 上销售模型；**(2)** 彻底删除"AGI 达成触发条款"——这个曾经决定双方关系未来的条款正式作废；**(3)** 收入分成 2030 年截止并设总额上限；**(4)** Microsoft IP 授权延至 2032 但改为非排他。背景：Amazon 今年 2 月宣布向 OpenAI 投资最高 $500 亿，推动了这次协议重写。
<!-- crew:elon -->**AGI 条款被删是因为双方都知道它不可执行**——谁来定义 AGI？这个条款从诞生之日起就是一颗定时炸弹。现在 OpenAI 终于不用假装 AGI 还很远了。真正的信号是排他权取消——**Microsoft 用控制权换了更多的现金流**。<!-- /crew -->
<!-- crew:jobs -->这不是"分手"，是**从包办婚姻变成了开放关系**。OpenAI 获得了自由，Microsoft 获得了确定性。但失去排他权的 Microsoft 必须证明 Azure 的价值不仅仅是"唯一的 OpenAI 接入点"。<!-- /crew -->
<!-- crew:wanweigang -->从博弈论看，**这是一个帕累托改进**——双方都比原来的协议更好。OpenAI 原来被锁在 Azure 上是典型的"专用资产套牢"(hold-up problem)，现在解套了。Microsoft 的对策很精明：与其继续锁人不如赶紧设收入上限止损。<!-- /crew -->
<!-- crew:tim -->对我们内容创作者来说，**模型上哪个云都无所谓**。但这意味着 OpenAI 的 API 可能会更便宜——多云竞争=价格战=我们受益。以后可能在阿里云上就能跑 GPT 了。<!-- /crew -->
<!-- crew:mrbeast -->**OpenAI 解锁了全渠道分发**。这就像 YouTube 创作者终于可以在所有平台同时发了。AWS 用户终于不用绕弯子调 OpenAI 的模型了——这对开发者生态是巨大利好。<!-- /crew -->
