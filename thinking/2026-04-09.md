## 🔍 今天的思考侧重：内容消费路径 × 社交传播 × 数据叙事

> 昨天的思考集中在 UI/UX 细节和技术债务，今天换一个更"产品策略"的角度：**用户是怎么找到这个网站、怎么看内容、看完之后又做什么？**

### 1. 用户旅程地图：三种典型路径

目前网站的用户（主要是 Nate 的 IEG 同事）大致有三条进入路径：

**路径 A：企微推送 → 点链接 → 浏览日报**
- 触发：每天收到企微群通知，看到"今日必看"摘要
- 行为：好奇 → 点开 → 快速扫 5 条必看 → 可能翻到感兴趣的分类 → 关闭
- 痛点：**入口是群聊消息，生命周期极短**——如果当时没点，消息很快被淹没。没有"稍后再看"机制

**路径 B：直接访问 → 查看最新日报**
- 触发：书签 / 记住了 URL / 浏览器历史
- 行为：打开首页 → 看今天的 → 可能翻翻昨天 → 离开
- 痛点：**没有任何"粘性机制"**——不会推送提醒"今天的日报已更新"，完全靠用户主动想起来

**路径 C：被同事口头安利 → 首次访问**
- 触发："你看过 Nate 那个 AI 新闻网站吗？"
- 行为：打开 → 看到最新一天的内容 → 不了解历史上下文 → 可能只看一次就忘了
- 痛点：**没有新手引导，也没有"最佳内容"入口**——新用户看到的永远是"今天碰巧有什么"，而不是"这个网站最精华的内容"

**产品机会**：
- 路径 A 可以通过**锚点链接**优化（昨天已提，P0 优先级）
- 路径 B 可以考虑 **PWA + Web Push**——用户授权后浏览器直接推送"今日更新"。实现成本不高（Service Worker + Push API），但体验提升巨大
- 路径 C 最需要一个 **"Best Of" / "Start Here" 页面**：精选过去一个月最值得看的 10 条新闻 + 几篇周报链接，给新访客一个"啊这个网站值得加书签"的第一印象

### 2. 社交传播的断裂点

**观察**：这个网站的内容质量是很高的（每条新闻都有来源、标签、关联分析），但它几乎**零社交传播能力**。

具体问题：
- **无分享按钮**：看到一条有意思的新闻，想转给同事或发朋友圈，没有快捷方式。用户只能复制整个页面 URL
- **无 Open Graph / Twitter Card 元标签**：在微信、企微、飞书等 IM 中粘贴链接，预览只会显示一个干巴巴的 `<title>`，没有描述、没有缩略图——完全丧失点击欲
- **无单条新闻的独立 URL**：即使想分享"三巨头反击蒸馏"这条新闻，也只能分享整天的日报链接，接收者还得自己找
- **企微推送只到群里**：如果有人不在那个群，就永远看不到。没有公开的订阅渠道（邮件/RSS）

**优化方向**：

#### 2a. OG Meta 标签（优先级：P0，工作量：小）
```html
<meta property="og:title" content="AI 新闻速递 - 2026-04-09">
<meta property="og:description" content="今日 21 条新闻：三巨头联手反击中国 AI 蒸馏 | DeepSeek 专家模式上线 | 国产芯片市占破 40%">
<meta property="og:image" content="https://nathanliu100.github.io/ai-news-collector/og-image.png">
<meta property="og:type" content="article">
```
动态 OG 在纯静态站点上有难度（GitHub Pages 不支持 SSR），但可以用一个固定的品牌 OG 图 + 通用描述作为基线。更高级的方案是用 GitHub Actions 每天生成当日的 OG 图。

#### 2b. 单条新闻锚点 + 分享按钮（优先级：P0，工作量：中）
- 每条 H3 新闻自动生成 `id`（昨天已提）
- 在每条新闻旁边加一个 🔗 小图标，点击复制 `URL#anchor` 到剪贴板
- 移动端长按可触发系统分享菜单（Web Share API）

#### 2c. RSS Feed（优先级：P2，工作量：小）
- 在 GitHub Actions 里额外生成一个 `feed.xml`
- 技术向的同事（尤其是做后端的）可能更喜欢 RSS 阅读器订阅
- 也方便未来接入其他平台（即刻、Telegram Bot 等）

### 3. 数据叙事：把"数据快照"变成视觉故事

**观察**：每天日报底部的「📊 数据快照」是一个纯文本表格，包含当日关键数据指标。但作为"快照"，它缺少**时间维度**——用户只能看到今天的数字，看不到趋势。

**连续 3 天数据对比（手动提取）**：

| 指标 | 4/7 | 4/8 | 4/9 | 趋势 |
|------|-----|-----|-----|------|
| 中国 AI 周调用量 | 4.69 万亿 | 12.96 万亿 | 12.96 万亿 | 📈 暴增 |
| Q1 全球创投 | $2970 亿 | $2970 亿 | — | 持平（同一数据点） |
| 可灵年化收入 | $2.4 亿 | — | — | 缺失 |
| DeepSeek 宕机次数 | — | — | 7 次 | 🆕 新指标 |

**问题一目了然**：
- 每天的数据指标**不一致**——不同日期跟踪的指标完全不同，无法纵向对比
- 没有可视化——纯数字表格在移动端体验很差（横向滚动）
- "变化"列是文字描述（"同比 +150%"），没有统一格式

**产品方向**：

#### 方案 A：固定指标仪表盘（简单但有效）
定义一组"核心跟踪指标"（比如 5-8 个），每天在新闻采集时更新。前端用简单的数字 + 箭头展示趋势：
```
中国 AI 周调用量   12.96T ▲176%
英伟达中国市占      55%   ▼
DeepSeek 月活      1.37亿  →
```
这比表格直观得多，而且在移动端可以做成横向滑动的 Stat Cards。

#### 方案 B：周度数据可视化（更有野心）
每周周报里嵌入一张 SVG 趋势图——不需要 Chart.js 那么重，纯 SVG 内联就行：
- X 轴：过去 7 天
- Y 轴：新闻数量、重大新闻占比、部门关联新闻数
- 一张迷你折线图，杂志风格，不要花哨

这能让周报的"数据快照"部分从"一段数字"变成**视觉叙事**，提升整份周报的品质感。

## 🔍 额外观察：内容重复度量化

> 连续读了 3 天的日报后，作为"每天都看的用户"，我有一个直觉性的感受需要量化。

**3 天内重复出现的主题**（出现 ≥2 天）：
1. AI 视频格局（可灵 3.0 / Seedance 2.0）→ 4/7, 4/8, 4/9 **连续 3 天**
2. DeepSeek V4 发布倒计时 → 4/7, 4/8, 4/9 **连续 3 天**
3. Q1 全球创投 $2970 亿 → 4/7, 4/8 **2 天**
4. AI 模型"小数点时代" → 4/7, 4/9 **2 天**
5. Qwen3.6-Plus 编程能力 → 4/7, 4/9 **2 天**

**重复率**：约 5/21 ≈ **24%** 的新闻是续报或高度相关内容。

**这不一定是坏事**——重大事件本来就需要持续跟踪。但展示方式需要优化：
- 方案 1：在续报新闻的标题前加 `🔄` 并链接到首报日期（**最小改动**）
- 方案 2：新闻采集 prompt 里增加去重指令——如果某条新闻和前 3 天高度重叠且无实质新信息，降级为一行摘要而非完整条目
- 方案 3：「故事线」视图（昨天的"大胆想法"）——这是终极方案但工作量大

**建议先做方案 2**：改采集 prompt 比改前端快，而且直接提升内容质量。

## 💡 今天的大胆想法

**个人化 AI 新闻摘要（"My Briefing"）**

场景：Nate 的团队有做设计的、做开发的、做策划的。他们关心的 AI 新闻维度很不一样。

设想一个功能：用户在网页上勾选自己关心的标签（比如 `AI+设计`、`AI+游戏`、`国内AI`），网站用 localStorage 记住偏好。之后每次打开：
- 默认只展开用户关心的分类
- "今日必看"区域也根据偏好重排序（`💎 部门相关` 标签的条目优先）
- 甚至生成一段个性化摘要："今天有 3 条和你关注领域相关的新闻"

**这不需要后端**——所有逻辑都在前端用 localStorage + 筛选就能实现。但它会让网站从"Nate 的新闻站"变成"每个人的 AI 新闻站"。

如果做得更极端：**配合 AI 生成个性化每日邮件摘要**。用户订阅 + 选偏好 → 每天自动生成定制版摘要推送到邮箱。这需要后端（但也可以用 GitHub Actions + Resend/SendGrid 免费额度实现）。
