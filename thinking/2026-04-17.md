### 2. "关我事"指数——一个全新的信息过滤维度

> 当前信息架构有 3 个过滤维度：重要性（🔴🟡🟢）、时效性（🆕🔄📌）、可信度（Tier 1/2/3）。缺少第 4 个：相关性。

#### 2a. 痛点分析

AI News Collector 的目标用户是 **IEG 游戏多媒体设计部门的同事**。但当前的新闻排序完全是"AI 行业视角"：

```
当前逻辑：
  Claude Opus 4.7 > Codex 升级 > Qwen3.6 > 混元 3D
  （按模型发布的影响力排序）

部门视角应该是：
  Anthropic AI 设计工具 > 混元 3D 2.0 > Opus 4.7 视觉能力 > 其他
  （按"对设计师日常工作的直接影响"排序）
```

**当前用 💎 标签来标注"部门相关"**——今天有 11 条新闻带 💎。但 💎 只是一个二元标签（相关/不相关），没有梯度。对设计师来说，混元 3D 2.0（可能明天就能用）和 Qwen3.6（要到代码层面才有意义）的"相关度"天差地别。

#### 2b. 解决方案思考

**方案 A：在今日必看中加入"部门特别关注"子section**

```markdown
## ⚡ 今日必看

1. 🔴 Claude Opus 4.7 正式发布 ...
2. 🔴 OpenAI Codex 重大升级 ...
3. 🔴 阿里开源 Qwen3.6-35B-A3B ...
4. 🔴 腾讯发布并开源混元 3D 世界模型 2.0 ...

### 💎 部门特别关注
- **Anthropic AI 设计工具** — 用自然语言生成网站和演示文稿，直接挑战 Figma/Adobe。设计流程可能被重新定义。
- **混元 3D 2.0** — 文字生成 3D 资产，直接导入 Unity/UE。3D 资产生产管线的潜在颠覆。
```

**优点**：不改变整体排序逻辑，只是在顶部加一段"对我们来说特别重要的"指引。
**工作量**：纯 prompt 修改，在 COLLECTOR_PROMPT 里加一条规则："如果今天有 3 条以上 💎 新闻，在今日必看末尾追加一个'💎 部门特别关注'子section，挑出对游戏多媒体设计最直接相关的 2-3 条，各写一句话说明和部门工作的关联"。15min。

**方案 B：企微推送分两段**

```
⚡ AI 行业头条（4条）
🔴 Claude Opus 4.7 正式发布
🔴 OpenAI Codex 重大升级
🔴 阿里开源 Qwen3.6-35B-A3B
🔴 腾讯发布混元 3D 世界模型 2.0

💎 和我们直接相关（2条）
🔴 Anthropic AI 设计工具正面挑战 Figma/Adobe — 设计流程可能被重新定义
🔴 混元 3D 2.0 — 文字→3D 资产→Unity/UE，就是我们在做的事

📰 完整日报 → link
```

**优点**：推送场景下的信息分层更清晰——同事可以只看第二段。
**工作量**：修改推送 prompt，20min。

**方案 C：前端增加"💎 部门相关"filter**

在 filter 按钮组（All / Critical / Notable / FYI）旁边加一个 💎 按钮，只显示带 💎 标签的新闻。

**优点**：用户自己选择"只看和我相关的"。
**工作量**：
- JS：在 `structureNewsBlocks()` 里检测 💎 emoji → `block.dataset.dept = 'true'`
- CSS：新增一个 `.pill` 按钮
- 约 30min

**建议**：先做 A + B（纯 prompt 层面，0 前端代码），效果验证后再考虑 C。

## 💡 今天的大胆想法

**"个性化视角生成器"——同一份新闻，不同角色看到不同的解读**

今天的核心发现是：同一条新闻（Anthropic AI 设计工具），AI 行业分析师看到的是"估值变动"，设计师看到的是"工具替代威胁"，前端工程师看到的是"竞品分析"。

设想：

```
AI Daily Briefing — 角色视角切换

[🌍 全局视角]  [🎨 设计师视角]  [💻 工程师视角]  [📊 PM视角]

当前：设计师视角 ✓

⚡ 今日必看（为你筛选）
1. 🔴 Anthropic AI 设计工具 — Figma/Adobe 股价跌 2%+，
   你的 Figma 工作流可能在 12 个月内被重新定义
2. 🔴 混元 3D 2.0 — 文字→3D→Unity/UE，
   3D 资产生产管线的颠覆者
3. 🟡 Opus 4.7 视觉分辨率 3 倍 — 
   AI 现在能"看懂"你的设计稿了
```

**为什么这个想法值得记录**：

1. AI News Collector 目前是"一份报纸给所有人"。但 Nate 部门的设计师和后台开发者关注点完全不同
2. 技术上，这可以纯靠 prompt 层面实现——为每个角色生成不同的"今日必看"排序 + 一句话解读
3. 不需要前端改动（可以作为不同的 Markdown 文件：`2026-04-17.designer.md`）
4. 企微推送可以分群发送：设计群收到设计师视角，开发群收到工程师视角

**落地路径**：
- V0：先在采集 prompt 里加一个"💎 部门特别关注"section → 今天提的方案 A
- V1：生成 `YYYY-MM-DD.designer.md` 额外文件，前端加 tab 切换
